LINUX WHISPER DICTATION: Product Requirements Document (Todo Format)
Tech Stack: Python CLI (Click/Typer), faster-whisper (CTranslate2), PipeWire/PulseAudio/ALSA, Groq/Gemini/OpenRouter APIs
Philosophy: CLI-first, cross-distro, local STT with free-tier LLM text sanitization
================================================================================

(A) Implement PulseAudio audio capture fallback using parec subprocess for distros without PipeWire +audio
(A) Implement ALSA audio capture fallback using arecord for minimal systems without PipeWire or PulseAudio +audio
(A) Write audio backend auto-detector that probes for PipeWire > PulseAudio > ALSA in order and selects first available +audio
(A) Implement WAV temp file writer that saves captured audio chunks to /tmp with automatic cleanup on exit +audio
(A) Add configurable audio input device selection by listing available capture devices per backend +audio
(A) Implement silence detection using RMS energy threshold to auto-stop recording after configurable silence duration (default 2s) +audio
(A) Add push-to-talk recording mode that captures audio only while global hotkey is held +audio
(A) Add toggle recording mode that starts/stops capture on single hotkey press +audio
(A) Write unit tests for each audio backend using mock subprocess calls to verify capture command construction +audio

(A) Implement OpenRouter backend using httpx POST to https://openrouter.ai/api/v1/chat/completions with free model selection +llm
(A) Write system prompt for text sanitization: fix punctuation, capitalization, filler words, grammar while preserving meaning +llm
(A) Add raw mode that skips LLM sanitization entirely and outputs whisper transcript as-is +llm
(A) Implement API key storage in ~/.config/linux-whisper-dictation/config.toml with per-backend key fields +llm

(A) Write integration tests for each LLM backend using recorded HTTP responses (VCR-style mocking) +llm
