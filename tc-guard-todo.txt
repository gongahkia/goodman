# TC-Guard: T&C Summarizer Browser Extension
# FOSS browser extension that auto-detects T&C checkboxes, summarizes legal text via LLM, flags concerning clauses
# Chrome-first (Manifest V3), Firefox compat, Safari stretch goal
# BYOK model support: Claude, OpenAI, Gemini, local models (Ollama)
# Zero telemetry, localStorage caching, TypeScript, popup overlay UI

# ============================================================
# +detection — T&C Element Detection Engine
# ============================================================
(A) Build MutationObserver wrapper that watches DOM for dynamically inserted checkboxes, modals, and banner elements +detection
(A) Implement checkbox detector: find <input type="checkbox"> elements adjacent to anchor tags or labels containing T&C keywords (terms, conditions, privacy, policy, agree, consent, EULA, TOS) +detection
(A) Build keyword dictionary with weighted scoring: exact matches ("terms and conditions") score higher than partial ("terms") +detection
(A) Implement proximity scorer: walk DOM tree upward/sideways from checkbox to find nearest T&C link/text within 5 ancestor levels +detection
(A) Handle label-for associations: resolve <label for="id"> to find checkbox-label pairs containing T&C references +detection
(A) Detect cookie consent banners by matching common class/id patterns (cookie-banner, consent-modal, gdpr) and button text patterns (accept, reject, manage) +detection
(A) Detect full-page T&C walls: identify pages where >60% visible text is legal boilerplate using keyword density heuristic +detection
(B) Detect privacy policy links in page footers and signup forms by scanning <a> tags with href containing "privacy" or "terms" +detection
(B) Detect iframe-embedded T&C content: scan iframes with src containing legal/terms paths, request content via background script +detection
(B) Implement debounced re-scan: re-run detection 500ms after DOM mutations settle to catch SPA route changes +detection
(B) Build confidence scoring system: each detected element gets a 0-1 confidence score based on keyword match + proximity + structural heuristics +detection
(B) Filter false positives: exclude elements inside nav bars, already-checked boxes, and elements with display:none +detection
(C) Detect T&C modals triggered by checkbox click: intercept click event, detect if a modal appeared within 1s, pause and scan modal content +detection

# ============================================================
# +extraction — Legal Text Extraction Pipeline
# ============================================================
(A) Build inline text extractor: given a detected T&C region, extract all visible text content from the container element and its descendants +extraction
(A) Implement link follower: for detected <a> tags pointing to T&C pages, fetch the linked page via background script fetch() and extract <main> or <article> or <body> text +extraction
(A) Build text cleaner: strip HTML tags, normalize whitespace, remove navigation/header/footer boilerplate from extracted legal text +extraction
(A) Implement character limit check: if extracted text < 200 chars, attempt to follow the nearest T&C hyperlink for full text +extraction
(B) Build PDF extractor: detect links ending in .pdf, fetch via background script, parse text content using pdf.js +extraction
(B) Implement chunking strategy: split legal text into ~3000 token chunks at paragraph boundaries for long documents +extraction
(B) Build text deduplication: hash extracted text sections to avoid re-processing identical T&C blocks across page navigations +extraction
(B) Handle multi-document T&C: when multiple links detected (Terms + Privacy + Cookie Policy), extract each separately and tag by type +extraction
(C) Extract T&C from iframe sources by messaging between content script and background script to bypass same-origin restrictions +extraction
(C) Detect and handle "click to expand" T&C sections: simulate expansion click, wait for content, then extract +extraction

# +summarizer — Summarization & Analysis Engine
# ============================================================
(A) Build system prompt for T&C summarization: instruct LLM to output structured JSON with fields {summary, key_points[], red_flags[], data_usage, user_rights, opt_outs[]} +summarizer
(A) Implement single-shot summarizer: for texts under 3000 tokens, send full text with system prompt in one LLM call +summarizer
(A) Implement chunked summarizer: for texts over 3000 tokens, summarize each chunk independently then run a merge pass to consolidate into final summary +summarizer
(A) Define red flag taxonomy with sensible defaults: data_selling, third_party_sharing, arbitration_waiver, class_action_waiver, auto_renewal, unilateral_changes, perpetual_license, location_tracking, biometric_data, no_deletion_right +summarizer
(A) Build red flag severity classifier: map each detected flag to severity (high/medium/low) based on keyword context from LLM output +summarizer
(B) Implement summary response validator: parse LLM JSON output, validate against expected schema, fallback to raw text display if malformed +summarizer
(B) Build summary caching layer: hash extracted text, store summary in chrome.storage.local keyed by hash, serve cached on re-encounter +summarizer
(B) Implement cache invalidation: expire cached summaries after 30 days or when T&C text hash changes +summarizer
(B) Add language detection: detect non-English T&C text, prepend "translate and summarize" instruction to prompt +summarizer
(C) Implement comparison mode: when cached summary exists but text changed, highlight what changed between old and new T&C +summarizer
