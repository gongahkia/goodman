### Task 5 (A) +audio
blockedBy: [1, 4]

**PURPOSE** — Resamples recorded audio to 16kHz mono, the exact format Whisper expects. Without this, model inference produces garbage.

**WHAT TO DO**
1. Add `rubato = "0.16"` to `Cargo.toml`.
2. In `src-tauri/src/managers/audio.rs`, implement `resample_to_16khz(samples: &[f32], source_rate: u32) -> Result<Vec<f32>>`:
   - If `source_rate == 16000`, return a clone.
   - Otherwise, use `rubato::SincFixedIn` with:
     - `resample_ratio`: `16000.0 / source_rate as f64`
     - `sinc_len`: 256
     // WHY 256: standard sinc interpolation length for high-quality audio resampling; shorter introduces aliasing, longer adds latency with diminishing returns
     - `f_cutoff`: 0.95
     // WHY 0.95: passes 95% of the Nyquist bandwidth; prevents aliasing while preserving nearly all speech-frequency content
     - `interpolation`: `SincInterpolationType::Linear`
     - `oversampling_factor`: 256
     // WHY 256: matches sinc_len for balanced quality/performance in offline resampling (not real-time constrained)
     - `num_channels`: 1
   - Process the samples in chunks matching `input_frames_max()`.
   - Concatenate output chunks into a single `Vec<f32>`.
3. Implement `save_wav(samples: &[f32], sample_rate: u32, path: &Path) -> Result<()>`:
   - Add `hound = "3.5"` to `Cargo.toml`.
   - Write a 16-bit PCM WAV file using `hound::WavWriter` with the given sample rate and 1 channel.
   - Convert f32 samples to i16 by multiplying by `i16::MAX as f32` and clamping.

**DONE WHEN**
- [ ] Resampling a 1-second 44100Hz sine wave (440Hz) to 16kHz produces a Vec of length ~16000 (±100).
- [ ] The resampled signal's dominant frequency is still ~440Hz (verify via zero-crossing count: expect ~880 zero crossings ±50 in 16000 samples).
- [ ] `save_wav` produces a valid WAV file readable by `hound::WavReader`.
