HAIR VISUALIZER - Product Requirements Document (Todo Format)
================================================================================
Project: Web-based Haircut Visualizer using Local Python CV & GenAI
Tech Stack: Python (FastAPI, OpenCV, MediaPipe, Diffusers), Frontend (React/Vite), Websockets
Scope: Real-time AR overlay (Core) + Local Generative AI Inpainting (Stretch)
Platform: Desktop Web (Localhost)
================================================================================

PROJECT SETUP & INFRASTRUCTURE
Initialize Git repository with standard .gitignore (Python/Node)
Create project directory structure (backend/, frontend/, models/, assets/)
Initialize frontend project using Vite + React + TypeScript
Setup Python virtual environment (venv/conda)
Install core CV dependencies (opencv-python, mediapipe, numpy, pillow)
Install backend server dependencies (fastapi, uvicorn, python-multipart, websockets)
Install GenAI dependencies (torch, diffusers, transformers, accelerate)
Create centralized config.py for app settings (camera index, model paths)
Setup logging configuration for backend services
Create README.md with comprehensive installation and run instructions

BACKEND - COMPUTER VISION (REAL-TIME)
Create camera_manager.py to handle OpenCV VideoCapture in a separate thread
Implement thread-safe frame reading mechanism to prevent I/O blocking
Integrate MediaPipe Face Mesh for high-fidelity 468-point landmark detection
Create face_processor.py to extract specific hair-relevant landmarks (forehead, temples, ears)
Implement distinct detection for head pose (yaw, pitch, roll) to adjust asset perspective
Create segmentation_utils.py using MediaPipe Selfie Segmentation to mask out original hair
Implement a robust background subtraction fallback if segmentation fails
Create a coordinate mapping function to normalize landmark points for the frontend
Develop a "FaceAnchor" class to calculate stable anchor points for hair attachment
Optimize frame processing loop to maintain consistent 30+ FPS
Implement JSON serialization for sending landmark data via WebSocket

BACKEND - API & SERVER
Initialize FastAPI application instance
Create WebSocket endpoint /ws/video for streaming processed frames or landmark data
Implement HTTP endpoint /api/hair-styles to serve list of available assets
Create static file mount for serving hair asset images
Implement /api/upload endpoint for users to upload their own reference photo
Create /api/capture endpoint to save high-res snapshots from the canvas
Implement error handling middleware for API and WebSocket connections
Add CORS configuration to allow frontend (port 5173) to talk to backend (port 8000)

FRONTEND - CORE ARCHITECTURE
Setup React Context for global state (CameraStatus, SelectedStyle, FaceData)
Create distinct layers in UI: VideoBackground, AROverlay, UIControls
Implement WebSocket client hook to receive real-time face landmarks
Create useCamera hook to manage webcam permission and stream lifecycle
Implement a synchronized Canvas layer that matches video feed dimensions
Create a coordinate system converter (Video Space -> Screen Space)

FRONTEND - AR/2D OVERLAY ENGINE
Create HairAsset component capable of rendering transparent PNGs
Implement Affine Transformation logic to warp hair assets based on face anchors
Map specific hair asset control points to Face Mesh landmarks (e.g., sideburns to ears)
Implement dynamic scaling of hair assets based on face depth (z-coordinate)
Create a rotation logic to tilt hair assets matching head roll
Implement opacity controls to blend hair asset with video feed
Add a manual "nudge" mode allowing users to fine-tune hair position (X, Y, Scale)
Implement "Flip Horizontal" toggle for mirroring hair styles
Create a masking composite operation to hide original hair (basic overlay)

ASSET MANAGEMENT & PREPARATION
Source 20 distinct hairstyle assets (Short, Medium, Long, Bangs, Buzz) with transparent backgrounds
Standardize all hair assets to a reference face template resolution
Create a metadata JSON file linking assets to their optimal anchor points
Pre-process assets to separate "front hair" (bangs) and "back hair" layers for depth
Create category tags for assets (e.g., "Business", "Casual", "Crazy")

UI/UX - INTERACTION DESIGN
Design a clean, modern sidebar for hairstyle selection
Implement grid view for hairstyle thumbnails
Create a floating control panel for manual adjustments (Color, Position, Size)
Implement a "Compare" slider to toggle between original and new look
Add "Snapshot" button with flash animation and gallery view
Create a "Loading" state for model initialization
Implement keyboard shortcuts for quick style switching (Arrow keys)

FEATURE - COLOR & TEXTURE
Implement CSS filter-based color tinting for hair assets (Sepia/Hue-Rotate)
Create a more advanced Canvas-based color multiplication for realistic dyeing
Add preset color palette swatches (Blonde, Brunette, Black, Red, Neon)
Implement a brightness/contrast slider to match lighting conditions
Create a "Color Picker" tool to sample colors from the video feed

STRETCH GOALS - LOCAL GEN AI (STABLE DIFFUSION)
Research and select a fast Inpainting model (e.g., Stable Diffusion Inpainting v1.5 or SDXL Turbo)
Create genai_manager.py to load the model into VRAM (optimized for 8GB+ cards)
Implement an endpoint /api/generate that accepts a base image and prompt
Create a prompt engineering helper to construct optimal prompts ("haircut, photorealistic")
Implement an automatic mask generator using MediaPipe hair segmentation
Create a "Generative Fill" UI mode separate from the AR mode
Implement "Img2Img" pipeline to guide the generation with the 2D overlay as a sketch
Add a "Strength" slider to control how much the AI deviates from the AR overlay
Implement a queue system for generation requests to prevent GPU OOM
Create a gallery to save and compare AI-generated variations
Implement "Seed" control for reproducible results
Add "Negative Prompt" configuration to remove unwanted artifacts (hats, glasses distortions)
Create a progress bar in UI polling the backend for generation status
Optimize model loading to be "on-demand" to save resources when not in use
Implement specific "Hair Texture" LoRA support for better realism (optional)
Create a "Refine" button to upscale and polish the best AI result
Add ability to export the generated image as PNG/JPG
Implement a "History" tab to view previous generations in the session
Create a "Side-by-Side" view to compare input frame vs AI result
Add a toggle to switch between CPU and GPU inference (with warning for CPU slowness)
Document hardware requirements for the GenAI features in README
Clean up VRAM resources when switching back to pure AR mode
Final integration test ensuring AR and GenAI modes coexist without crashing
