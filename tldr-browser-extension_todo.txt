(A) PROJECT: TL;DR Browser Extension PRD

-- PHILOSOPHY --
- User experience is paramount; the product should be unobtrusive and "just work".
- Anonymity is a core feature to encourage honest and frictionless contributions.
- Community moderation is essential for maintaining quality and trust.
- Start simple, build a strong foundation, and then expand.

-- EPIC: Project Foundation & Setup --
(A) Initialize project with a Git repository and a clear README.md
(A) Choose and configure a bundler (e.g., Vite, Webpack) for browser extension development.
(A) Create the initial `manifest.json` for a Chrome extension (Manifest V3).
(A) Set up project structure: folders for backend, extension (src), and docs.
(A) Define a simple, memorable name and create placeholder logo assets.
(A) Set up linter and code formatter (e.g., ESLint, Prettier) to ensure code quality.

-- EPIC: Core Backend Service --
(A) Design the database schema: `pages` (by URL), `summaries`, `votes`, `anonymous_users`.
(A) Implement a system to generate and assign a persistent unique ID for each anonymous user (store in browser).
(A) Create backend API server using a lightweight framework (e.g., Node.js/Express, FastAPI).
(A) API Endpoint: `GET /api/summaries?url={page_url}` to fetch all summaries for a page.
(A) API Endpoint: `POST /api/summaries` to submit a new summary (linked to anonymous_user_id).
(A) API Endpoint: `POST /api/votes` to cast an upvote or downvote for a summary.
(A) API Endpoint: `GET /api/user/{anonymous_user_id}` to check user's status/reputation.
(B) Implement basic rate limiting to prevent API abuse.

-- EPIC: Browser Extension - Core UX (Chrome MVP) --
(A) Develop content script to inject a small, non-intrusive UI element onto pages (e.g., floating icon).
(A) Ensure the injected UI element is dismissible for the session or permanently for that site.
(A) Design and build the main summary panel/overlay, triggered by clicking the UI element.
(A) The summary panel should be clean, easy to read, and feel lightweight.
(A) Implement the view for listing all community-submitted summaries.
(A) Develop the UI for upvoting and downvoting summaries within the panel.
(A) Create a simple, clean form within the panel for submitting a new TL;DR.
(A) The submission form should include a character counter to enforce brevity (e.g., ~280 chars).
(A) Ensure all communication between the extension and the backend is done securely (HTTPS).
(C) Gracefully handle pages with no summaries (show a "Be the first to add one!" message).

-- EPIC: Community Moderation & Ranking --
(B) Develop the summary ranking algorithm: prioritize by net upvotes, but consider user reputation and recency.
(B) Implement the backend logic for calculating and storing user reputation scores.
(B) A user's reputation increases with upvoted summaries and decreases with downvoted ones.
(B) Add a "flag" or "report" button to each summary in the UI.
(B) Create an API Endpoint: `POST /api/reports` to submit a report against a summary.
(B) Implement a backend mechanism to track the number of flags on a summary.
(B) Automatically hide summaries that pass a certain flag threshold, pending admin review.
(C) Visually distinguish summaries from users with high reputation scores (e.g., a small badge).

-- EPIC: Admin & Official Summaries --
(B) Create a simple, password-protected web interface for admin tasks.
(B) Admin dashboard should list all summaries that have been flagged for review.
(B) Admins must be able to view, approve, or delete flagged summaries.
(B) Implement the admin functionality to mark a specific summary as "official".
(B) Official summaries should be pinned to the top of the summary list and have a distinct visual style.
(B) Admins should have the ability to ban/de-platform a malicious anonymous_user_id.

-- EPIC: V2 & Future Expansion --
(C) Research and document the process for porting the extension to Firefox.
(C) Research and document the process for porting the extension to Safari.
(C) Scope a potential "v2" feature: allowing users to optionally create accounts to sync reputation across devices.
(C) Investigate adding support for summarizing video content (e.g., YouTube) via community submissions.
(C) Consider a feature for users to see their own contribution history (tied to their anonymous ID).

-- EPIC: Deployment & Launch --
(A) Write clear, concise user-facing documentation on how the extension works.
(B) Set up a simple CI/CD pipeline to automate the extension build process.
(B) Prepare all required assets and descriptions for the Chrome Web Store.
(A) Perform thorough testing on a variety of popular websites.
(B) Submit the extension to the Chrome Web Store for review.
(C) Create a simple landing page to explain the product and link to the store page.
